🧱 1. DATABASE STRUCTURE (MANDATORY)
--------------------------------------------------

Create the following tables:

1. IMPORT LOGS TABLE

CREATE TABLE import_logs (
    id INT AUTO_INCREMENT PRIMARY KEY,
    search_query VARCHAR(255),
    category VARCHAR(100),
    city VARCHAR(100),
    records_fetched INT,
    source ENUM('google','manual','user'),
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

--------------------------------------------------

2. SEARCH LOGS TABLE

CREATE TABLE search_logs (
    id INT AUTO_INCREMENT PRIMARY KEY,
    search_query VARCHAR(255),
    category VARCHAR(100),
    city VARCHAR(100),
    results_found INT,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

--------------------------------------------------

3. BUSINESS STATS TABLE

CREATE TABLE business_stats (
    id INT AUTO_INCREMENT PRIMARY KEY,
    business_id INT,
    views INT DEFAULT 0,
    clicks INT DEFAULT 0,
    calls INT DEFAULT 0,
    whatsapp_clicks INT DEFAULT 0,
    last_updated TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

--------------------------------------------------

4. UPDATE BUSINESS TABLE

ALTER TABLE businesses ADD:
source ENUM('google','manual','claimed') DEFAULT 'google',
imported_at DATETIME,
last_updated DATETIME;

⚙️ 2. TRACKING LOGIC (CORE IMPLEMENTATION)
--------------------------------------------------

A. WHEN USER SEARCHES

- Log search into search_logs
- Store:
  search_query, category, city, results_found

If results_found = 0:
- Mark as HIGH DEMAND query

--------------------------------------------------

B. WHEN GOOGLE API IS CALLED

- Count fetched records
- Insert into import_logs

Example:
INSERT INTO import_logs 
(search_query, category, city, records_fetched, source)

--------------------------------------------------

C. WHEN BUSINESS IS VIEWED

- Increment views in business_stats

--------------------------------------------------

D. WHEN USER CLICKS:

- Call button → increment calls
- WhatsApp → increment whatsapp_clicks
- View details → increment clicks
